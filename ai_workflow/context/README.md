# 渐进式上下文

> 原则：不要一次性把整个代码库塞进对话。先给「索引/结构」，再按当前步骤提供「刚好需要的」文件或片段，从而**大幅降低 token、减少遗忘**。

---

## 使用方式

- 对话**开始时**：只给 `context/_template_progressive_context.md` 里「第 1 层」的内容（项目结构、Spec 路径、当前任务名）。
- 对话**进行中**：每步只贴本步用到的文件或片段（见模板中的「按步提供」）。
- 需要**细节时**：再贴具体实现、报错信息、相关测试。

---

## 与 Workflow / Skill 的关系

- Workflow 的每一步可注明「本步需要的上下文」，对应本模板的「按步提供」。
- Skill 本身是「按需拉取」的上下文单元：先给 Skill 列表，再在用到时拉取具体 Skill 内容。
