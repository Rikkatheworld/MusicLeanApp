# 渐进式上下文清单（模板）

> 按「层」在对话中提供信息，避免首条消息塞满整个项目。复制到项目或需求下，按需勾选/填写。

---

## 第 1 层：对话开始时（只给这些）

- [ ] **项目/需求名称**：
- [ ] **项目根目录结构**（仅顶层，10 行内）：
  ```
  例如：/app, /server, /shared, /docs, /ai_workflow
  ```
- [ ] **本任务/需求一句话**：
- [ ] **要遵循的 Spec 路径**：（如 `docs/spec_project.md`、`spec/requirement/xxx.md`）
- [ ] **要执行的 Workflow 路径**（若有）：（如 `ai_workflow/workflow/xxx/WORKFLOW.md`）
- [ ] **本任务涉及的模块/目录**：（如 `composeApp/src/commonMain`、`server/src/main`）

**不要在这一层贴**：完整文件内容、长列表、整份代码库。

---

## 第 2 层：按步骤提供（当前步才贴）

- [ ] **当前步骤名**：（与 WORKFLOW 的 Step 或 STEP_xx 对应）
- [ ] **本步需要的文件路径**：（只列路径，或贴 1–3 个关键文件内容）
- [ ] **本步用到的 Skill/规约**：（若需，贴 Skill 全文或关键段落）
- [ ] **本步输入/上一步产物**：（若依赖上一步，只贴必要片段）

**原则**：本步用不到的文件不贴；大文件只贴相关片段（函数/类/配置块）。

---

## 第 3 层：问题排查时才贴

- [ ] **报错信息**（完整）
- [ ] **相关代码片段**（围绕报错位置）
- [ ] **环境/版本**（若与问题相关）

---

## 本项目/需求的「上下文约定」（可选）

（填写本项目你与 AI 的约定，例如：）

- 先读 `docs/spec_project.md` 再改代码。
- 改 `composeApp` 时只贴 `commonMain` 与当前平台 `androidMain`/`jvmMain`，不贴整工程。
- 新增 API 时先贴 `server` 的路由定义，再贴实现文件。
